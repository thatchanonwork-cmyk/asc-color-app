# ASC Annual Meeting — Employee Color Group Lookup

เว็บแอปสำหรับพนักงานกรอกรหัสพนักงานเพื่อดูกลุ่มสีของตัวเอง (สไตล์ boarding pass) พร้อมหน้า Admin Dashboard สำหรับทีมงานดูสรุปจำนวนแต่ละสีและสถานะเช็คอิน

## โครงสร้างไฟล์

```
asc-color-app/
├── server.js              # backend (Express)
├── package.json
├── data/
│   ├── employees.json     # ข้อมูลพนักงาน 293 คน (แปลงจากไฟล์ Excel ที่ให้มา)
│   └── checkins.json      # เก็บสถานะเช็คอิน (อัปเดตอัตโนมัติเมื่อพนักงานกรอกรหัส)
└── public/
    ├── index.html         # หน้าพนักงาน (กรอกรหัส → แสดงสี)
    └── admin.html          # หน้า Admin Dashboard
```

## ฟีเจอร์

- พนักงานกรอกรหัสพนักงาน → เว็บแสดงชื่อ, ตำแหน่ง, สาขา, ทีม, กลุ่มสี พร้อมข้อความเตือนให้แคปหน้าจอไว้ยืนยันรับของ
- ระบบนับ "เช็คอินแล้ว" อัตโนมัติทันทีที่พนักงานกรอกรหัสครั้งแรก (กรอกซ้ำไม่นับซ้ำ)
- หน้า Admin (`/admin.html`) ต้องใส่รหัสผ่านก่อนเข้า แสดง:
  - จำนวนพนักงานทั้งหมด, จำนวนเช็คอินแล้ว, % เช็คอิน
  - สรุปแยกตามกลุ่มสี (จำนวน/เช็คอินแล้ว/เปอร์เซ็นต์) พร้อม progress bar
  - ตารางค้นหา พร้อม filter ตามสี/สถานะเช็คอิน และปุ่มเช็คอิน/ยกเลิกด้วยมือ (เผื่อกรณีพนักงานไม่มีมือถือ ทีมงานช่วยกรอกแทนได้)

## รันทดสอบในเครื่องตัวเอง (ต้องมี Node.js ติดตั้งแล้ว)

```bash
cd asc-color-app
npm install
npm start
```

เปิดเบราว์เซอร์ไปที่:
- หน้าพนักงาน: http://localhost:3000
- หน้า admin: http://localhost:3000/admin.html (รหัสผ่านเริ่มต้น: `hrbpasc2026`)

**แนะนำให้เปลี่ยนรหัสผ่าน admin ก่อนใช้งานจริง** โดยตั้งค่าตัวแปรแวดล้อม `ADMIN_PASSWORD` (ดูวิธีด้านล่าง)

## วิธี Deploy ให้มี URL จริงใช้กับ QR Code (แนะนำ Render.com — ฟรี)

ไม่ต้องเขียนโค้ดเพิ่ม ทำตามขั้นตอนนี้:

1. **สร้างบัญชี GitHub** (ถ้ายังไม่มี) แล้วสร้าง repository ใหม่ เช่นชื่อ `asc-color-app`
2. **อัปโหลดโฟลเดอร์ `asc-color-app` ทั้งหมด** ขึ้น repository นั้น (ลากไฟล์ทั้งหมดใส่ผ่านหน้าเว็บ GitHub ได้เลย ไม่ต้องใช้ git command line)
3. ไปที่ [render.com](https://render.com) → สมัครสมาชิก (login ด้วย GitHub ได้เลย)
4. กด **New +** → **Web Service**
5. เลือก repository `asc-color-app` ที่อัปโหลดไว้
6. ตั้งค่า:
   - **Name**: asc-color-app (หรือชื่อที่ต้องการ)
   - **Region**: Singapore (ใกล้ไทยที่สุด)
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: Free
7. เพิ่ม Environment Variable (กด **Advanced** หรือ **Environment**):
   - Key: `ADMIN_PASSWORD` → Value: (ตั้งรหัสผ่านที่ต้องการ เช่น `ASCteam2026!`)
8. กด **Create Web Service** รอสักครู่ (2-3 นาที) ระบบจะ build และ deploy ให้อัตโนมัติ
9. เมื่อเสร็จจะได้ URL แบบ `https://asc-color-app.onrender.com`
   - หน้าพนักงาน: `https://asc-color-app.onrender.com`
   - หน้า admin: `https://asc-color-app.onrender.com/admin.html`
10. **สร้าง QR Code** ที่ชี้ไปยัง URL หน้าพนักงาน (ใช้เว็บสร้าง QR ฟรี เช่น qr-code-generator.com) แล้วนำไปพิมพ์/ติดหน้างาน

> หมายเหตุ: Render free tier จะ "หลับ" เมื่อไม่มีคนใช้งานนาน ๆ (sleep หลัง 15 นาที) พอมีคนเข้าครั้งแรกจะใช้เวลาปลุกเครื่องประมาณ 30-50 วินาที ถ้ากังวลเรื่องนี้ในวันงานจริง แนะนำอัปเกรดเป็น paid plan ราคาประมาณ $7/เดือน หรือเปิดเว็บทิ้งไว้/กดเข้าเว็บล่วงหน้าก่อนงานเริ่ม 5 นาที

## การอัปเดตข้อมูลพนักงานภายหลัง (ถ้ามีคนเพิ่ม/แก้ทีมสี)

แก้ไฟล์ `data/employees.json` โดยตรง (เป็น array ของ object) แล้ว deploy ใหม่ หรือแจ้งให้ผมช่วย regenerate จากไฟล์ Excel เวอร์ชันใหม่ได้

## ส่วนที่ยังไม่ได้ทำ (ตามที่คุยกัน)

- **ยังไม่มีระบบส่งอีเมล Boarding Pass** เพราะไฟล์ข้อมูลยังไม่มีคอลัมน์อีเมลพนักงาน — ตอนนี้ใช้วิธีให้พนักงานแคปหน้าจอผลลัพธ์แทน (มีข้อความเตือนในหน้าเว็บแล้ว)
- ถ้าภายหลังมีอีเมลพนักงานครบ แจ้งได้ ผมจะเพิ่มปุ่ม "ส่ง Boarding Pass เข้าอีเมล" ให้ (ต้องต่อกับบริการส่งอีเมล เช่น Gmail API หรือ SendGrid)
